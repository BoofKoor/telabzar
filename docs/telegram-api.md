# Telegram Bot API — Recent Changelog (project-relevant)

The `docker-compose.yml` pins the local Bot API server image to **`aiogram/telegram-bot-api:10.2`**,
so this project targets **Bot API 10.2** (latest as of 2026-07). Only changes that matter to Telabzar
are summarized below (uploads / local server, media sending, rich messages, callbacks/formatting).
Full official log: https://core.telegram.org/bots/api-changelog (versions/dates cross-checked with the
sources listed per entry). Do not edit this file from memory — re-read the sources.

## Baseline: Local Bot API server + large uploads (pre-10.x, still current)
Running your own Bot API server (`tdlib/telegram-bot-api`) lifts the 50 MB download / 20 MB upload
limits to ~**2000 MB**, serves files from local disk, and allows local file paths — this is the
foundation of Telabzar's card/gateway model, not a recent change.
Source: https://github.com/tdlib/telegram-bot-api

## 10.0 — 2026-05-08
Bot-to-bot communication and guest mode (bots receiving/replying in chats they aren't a member of);
also live photos, poll media, message-reaction permissions, and bot access settings.
- New: `User.supports_guest_queries`; `Message.guest_bot_caller_user`, `guest_bot_caller_chat`,
  `guest_query_id`; class `SentGuestMessage`; method `answerGuestQuery`.
- **Relevance:** none used by Telabzar today; noted for completeness.
- Sources: https://github.com/python-telegram-bot/python-telegram-bot/issues/5228 ·
  https://gramio.dev/changelogs/2026-05-31

## 10.1 — 2026-06-11  ★ used
Introduced **Rich Messages** — document-grade, block-structured formatting for bots.
- New: `sendRichMessage`, `sendRichMessageDraft`; `rich_message` parameter on `editMessageText`.
- **Relevance:** Telabzar delivers multi-item / rich posts using the Rich Message API
  (`app/tasks_download.py` imports `InputRichMessage`, `InputRichBlockParagraph`,
  `InputRichBlockPhoto`, `InputRichBlockSlideshow`, `InputRichBlockVideo`), with a plain
  album/caption fallback when `DL_RICH_POSTS` is off.
- Sources: https://github.com/python-telegram-bot/python-telegram-bot/issues/5261 ·
  https://www.techtimes.com/articles/318257/20260611/telegram-desktop-69-bots-get-document-grade-formatting-guardian-controls-streaming.htm

## 10.2 — 2026-07-14  ★ target
Adds new outgoing **rich block types** and ephemeral-message plumbing.
- New rich blocks: lists, tables, details, collages, slideshows, media blocks, thinking blocks;
  `InputRichMessage.blocks` for block-based outgoing rich messages.
- Also: `BotCommand.is_ephemeral`; `Message.receiver_user`, `Message.ephemeral_message_id`;
  `receiver_user_id` and `callback_query_id` when sending messages/media;
  `ReplyParameters.ephemeral_message_id`.
- **Relevance:** the rich-block set is the ceiling for how far rich-post delivery can go; the
  planned "button/text settings" update (custom text, HTML formatting, premium emoji) builds on the
  Rich Message + entity/formatting capabilities available at this version.
- Sources: https://github.com/NousResearch/hermes-agent/issues/64497 ·
  https://core.telegram.org/bots/api-changelog
